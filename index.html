Test Coverage

The following tests from MessageRouterTest.java are run in the CI pipeline:

  • Basic subscribe and publish functionality
  • Multiple subscribers receive the same message
  • Concurrent access safety
  • Publishing to channel with no subscribers
  • Subscription lifecycle management
  • Error cases and validation
  • Valid channel names
  • Router close behavior
  • Performance test with many subscribers

Run Locally with Docker

If you prefer to run the tests on your own machine, you can use the provided Dockerfile. This will compile the code, run the entire test suite, and build the final image.

Prerequisites: Docker Desktop must be installed and running.

  1. Clone this repository to your local machine.
  2. Open your terminal and navigate to the root of the project (the same folder as Dockerfile and pom.xml).
  3. Run the following command:
docker build -t message-router .

The test results will be printed to your terminal. If the build succeeds, all tests have passed.
